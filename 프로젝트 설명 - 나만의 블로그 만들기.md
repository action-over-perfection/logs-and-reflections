# 나만의 블로그 만들기

완벽주의와 이에 따른 두려움을 타파하는 프로젝트를 하기로 결심한 후, 그리고 초기 기획에서 큰 효과를 보지 못한 후 (참고 : [기획을 중간에 바꾸게 된 이유](https://github.com/action-over-perfection/logs-and-reflections/blob/main/기획을%20중간에%20바꾸게%20된%20이유.md)) 그렇다면 어떻게 프로젝트 목표를 달성할 것인지 고민하는 시간을 가졌습니다.<br>
어쩌면 당연하게도, 제가 가장 해보고 싶었던 것에 눈길이 가더라고요. 백엔드 개발을 시작하기로 마음먹자마자 우아한테크코스에 지원했었고, 프리코스 오픈미션은 백엔드를 본격적으로 시작하기에 완벽한 기회였습니다.<br>
백엔드에 입문하기 위해 어떤 내용을 학습해야할지 찾아보니 많은 곳에서 게시판 CRUD API를 기본 중 기본 프로젝트로 꼽는 것 같았습니다. 저는 이 게시판을 실제로 사용하고 싶기에 **기술 블로그**의 형태로 만들어가기로 했습니다.

### 목표
* 블로그를 웹페이지 형태로 제공하기
* 블로그에서 우아한테크코스 프리코스 기간에 작성한 학습 및 회고글을 볼 수 있도록 하기 
* 기능 및 코드를 발전시키면서 만들 수 있는 가장 최선의 블로그를 완성하기

### 작업 과정에서의 제약 사항
* 정보 탐색 시 블로그보다 **공식문서**를 활용하기
  * 게시판 형태의 api는 간단한 검색만으로도 구현 방법을 쉽게 알아낼 수 있더라고요.
  * 궁금할 때 공식문서를 제일 먼저 찾는 습관을 들이고 싶었기도 하고, 명확한 이유를 가지고 여러가지 방법들을 적용해보고 싶었기에 블로그를 참고하는 것을 최대한 지양하려고 합니다.

### 주요 기능
* 기본 기능
  * 글 게시, 수정, 삭제
  * 글 목록 조회
  * 글 상세 조회
* 마크다운을 활용하여 작성할 수 있는 글

### 기술 스택
* Java, Spring Boot
* Spring Data JPA
* MySQL
* Docker
